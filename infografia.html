Guía Completa: Bitbucket y Git

Desde la creación del repositorio hasta subir tu proyecto

Proceso Paso a Paso

Crear cuenta en Bitbucket

Si no tienes una cuenta, regístrate en bitbucket.org. Es gratis y te permite crear repositorios privados ilimitados.

Nota: Puedes registrarte con tu correo de Google, Microsoft o Apple para mayor facilidad.

Crear un nuevo repositorio

Una vez dentro de Bitbucket, haz clic en el botón "Create" y selecciona "Repository".

Workspace: group-composers
Nombre del repositorio: composers
Descripción: Proyecto web del equipo COMPOSER
Tipo: Privado (recomendado para trabajo en equipo)

Importante: No marques la opción "Include a README" si ya tienes archivos locales para subir.

Agregar colaboradores

En la configuración del repositorio, ve a "User and group access" y agrega a los miembros de tu equipo.

1. Busca por nombre de usuario o email
2. Asigna permisos: Write (pueden subir cambios) o Admin (control total)
3. Confirma la invitación

Instalar Git en tu computadora

Descarga Git desde git-scm.com e instálalo siguiendo las instrucciones para tu sistema operativo.

Windows: Descarga el instalador ejecutable
macOS: Usa Homebrew (brew install git) o descarga directa
Linux: sudo apt install git (Ubuntu/Debian)

Configurar Git por primera vez

Abre tu terminal (Linux/macOS) o Git Bash (Windows) y configura tu información:

git config --global user.name "Tu Nombre"
git config --global user.email "tu.email@ejemplo.com"

Nota: Usa el mismo email que registraste en Bitbucket para que se vinculen correctamente tus commits.

Preparar tu proyecto local

Organiza tus archivos en una carpeta con la siguiente estructura recomendada:

composers-project
index.html
index1.html
index2.html
index3.html
index4.html
images
foto1.jpg
foto2.jpg
foto3.jpg
css
styles.css

Inicializar Git en tu proyecto

Abre la terminal, navega a tu carpeta del proyecto y ejecuta:

cd /ruta/a/tu/proyecto
git init
git add .
git commit -m "Primer commit: versión inicial del proyecto"

Nota: El comando git add . agrega todos los archivos. Si quieres agregar específicos, usa git add nombre_archivo.

Conectar con el repositorio remoto

Conecta tu repositorio local con el que creaste en Bitbucket:

git remote add origin https://bitbucket.org/group-composers/composers.git

Importante: Asegúrate de usar la URL correcta de tu repositorio. Puedes encontrarla en la página principal de tu repositorio en Bitbucket.

Subir tu código a Bitbucket

Ejecuta el comando para subir tu código al repositorio remoto:

git push -u origin main

Nota: Si usas una versión antigua de Git, la rama por defecto podría llamarse "master" en lugar de "main".

¡Éxito! Si todo va bien, verás un mensaje confirmando que tu código se ha subido correctamente.

Verificar en Bitbucket

Visita tu repositorio en Bitbucket para confirmar que todos los archivos se hayan subido correctamente.

Deberías ver:

- Todos tus archivos listados
- El commit que hiciste con el mensaje "Primer commit"
- La rama main/master con todo tu código
