Introducción

Esta evaluación corresponde al desarrollo del examen parcial de la asignatura de Programación, en el cual se busca aplicar los conocimientos adquiridos sobre algoritmos de aprendizaje supervisado, específicamente el Perceptrón.

A lo largo de la actividad, el equipo seleccionó un dataset real, realizó un proceso de preparación de datos y construyó tres modelos distintos de Perceptrón, con el fin de analizar cómo cambios en la tasa de aprendizaje y en la selección de características impactan en el rendimiento del modelo.

Implementación - Perceptrón Básico

Clase PerceptronBasic
class PerceptronBasic:
    def __init__(self, learning_rate=0.01, n_epochs=1000):
        self.learning_rate = learning_rate
        self.n_epochs = n_epochs
        self.weights = None
        self.bias = None
        self.errors = []
        self.accuracy_history = []

    def activation_function(self, x):
        return 1 if x >= 0 else 0

    def predict(self, X):
        predictions = []
        for i in range(len(X)):
            linear_output = np.dot(X[i], self.weights) + self.bias
            predictions.append(self.activation_function(linear_output))
        return np.array(predictions)

    def evaluate(self, X, y):
        predictions = self.predict(X)
        accuracy = np.mean(predictions == y)
        return accuracy, predictions

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.weights = np.random.normal(0, 0.01, n_features)
        self.bias = 0
        
        print("Iniciando entrenamiento...")
        for epoch in range(self.n_epochs):
            total_error = 0
            correct_predictions = 0
            
            # Mezclar datos
            indices = np.random.permutation(n_samples)
            X_shuffled = X[indices]
            y_shuffled = y[indices]
            
            for i in range(n_samples):
                linear_output = np.dot(X_shuffled[i], self.weights) + self.bias
                prediction = self.activation_function(linear_output)
                error = y_shuffled[i] - prediction
                
                if error != 0:
                    self.weights += self.learning_rate * error * X_shuffled[i]
                    self.bias += self.learning_rate * error
                    total_error += abs(error)
                
                if prediction == y_shuffled[i]:
                    correct_predictions += 1
            
            epoch_error = total_error / n_samples
            epoch_accuracy = correct_predictions / n_samples
            
            self.errors.append(epoch_error)
            self.accuracy_history.append(epoch_accuracy)
            
            if (epoch + 1) % 200 == 0:
                print(f"Época {epoch + 1}: Error = {epoch_error:.4f}, Accuracy = {epoch_accuracy:.4f}")

Conclusiones

El desarrollo de los tres modelos de Perceptrón permitió comprender de forma práctica cómo los parámetros del aprendizaje supervisado influyen directamente en la capacidad predictiva del modelo.
El Modelo 1 (Básico) sirvió como punto de partida para observar el comportamiento estándar del algoritmo, evidenciando una precisión moderada y un aprendizaje estable pero lento.
El Modelo 2 (Modificado), con una tasa de aprendizaje más alta y un decaimiento progresivo, logró una mayor velocidad de convergencia, mejorando el equilibrio entre precisión y recall.
El Modelo 3 (Variación de Características) demostró que la selección de variables tiene un impacto significativo en el rendimiento, evidenciando la importancia del análisis de relevancia de características.

Recomendaciones

Optimización

Evaluar más combinaciones de tasas de aprendizaje y épocas para encontrar configuraciones óptimas.

Arquitectura

Implementar funciones de activación no lineales o perceptrones multicapa (MLP) para conjuntos complejos.

Características

Realizar selección de características más rigurosa con técnicas estadísticas o algoritmos especializados.

Datos

Ampliar la base de datos para mejorar la generalización del modelo y evitar sobreajuste.

Métricas

Complementar con métricas visuales (gráficos ROC, curvas Precision-Recall) para evaluación completa.

Interfaz

Integrar interfaz gráfica o web para ingreso de datos y visualización intuitiva de resultados.
