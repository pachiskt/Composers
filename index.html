1 Introducción y Objetivo

En la actualidad, el desarrollo de soluciones tecnológicas representa una herramienta clave para optimizar procesos, mejorar la experiencia de los usuarios y resolver problemas que afectan el desempeño en distintos ámbitos.

El objetivo de este trabajo es presentar el proceso inicial de identificación de un problema susceptible de resolverse mediante una aplicación desarrollada en Python, estableciendo las bases para su posterior diseño, desarrollo e implementación.

2 Perfil de Usuario

Descripción General: Plataforma educativa pensada para estudiantes de todo el mundo, desde escuelas hasta universidades.

A quién está dirigido:

  • Estudiantes de escuelas que buscan reforzar sus conocimientos básicos
  • Estudiantes de colegios que desean apoyo para sus tareas y proyectos
  • Estudiantes de universidades que requieren soluciones avanzadas

Qué ofrecemos: Recursos interactivos, soluciones personalizadas, herramientas de apoyo en diversas áreas y espacios de colaboración.

3 Problema Identificado

La falta de herramientas accesibles para organizar sus finanzas personales genera dificultades que impactan no solo en su bienestar económico, sino también en su rendimiento académico y emocional.

Soluciones Propuestas

1. Integración de pagos en el carnet universitario

Ventaja: Comodidad de tener identificación y transacciones en un solo medio.

Limitación: Requiere inversiones en infraestructura y acuerdos institucionales.

2. Plataforma de educación financiera

Ventaja: Fortalece la alfabetización financiera de los jóvenes.

Limitación: Adopción depende de la motivación del estudiante.

3. Gestor de Finanzas Personales (Seleccionada)

Ventaja: Práctica, responde a necesidad inmediata, aprovecha billeteras digitales como Yape y Plin.

Implementación: Factible en corto plazo y escalable con funciones adicionales.

4. Red de Beneficios y Descuentos

Ventaja: Impacto económico positivo al reducir gastos cotidianos.

Limitación: Requiere negociación constante con empresas externas.

import tkinter as tk
from tkinter import messagebox

class Gasto:
    def __init__(self, categoria, monto):
        self.categoria = categoria
        self.monto = monto

class GestorFinanzas:
    def __init__(self, presupuesto=0):
        self.gastos = []
        self.presupuesto = presupuesto
    
    def agregar_gasto(self, categoria, monto):
        gasto = Gasto(categoria, monto)
        self.gastos.append(gasto)
    
    def total_gastos(self):
        return sum(gasto.monto for gasto in self.gastos)
    
    def verificar_presupuesto(self):
        total = self.total_gastos()
        if self.presupuesto > 0:
            if total > self.presupuesto:
                return "⚠ Has superado tu presupuesto!"
            else:
                return f"Aún tienes disponible: S/{self.presupuesto-total:.2f}"
        else:
            return "No has establecido un presupuesto."
